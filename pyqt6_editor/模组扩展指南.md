# TACZ Lua Map 编辑器模组开发指南

## 1. 模组系统概述

### 1.1 什么是模组？

TACZ Lua Map 编辑器的模组是用户自定义的节点扩展包。通过编写 Python 脚本，你可以：
- 创建新的节点类型
- 扩展节点库
- 实现自定义功能
- 定制主题和样式

### 1.2 模组文件结构

```
pyqt6_editor/
├── mods/              # 模组目录
│   ├── my_mod.py      # 自定义模组文件
│   └── ...
├── node.py            # 节点基类
├── registry.py        # 节点注册表
└── editor.py          # 编辑器主类
```

## 2. 快速开始

### 2.1 安装模组

1. 打开 `pyqt6_editor/mods/` 目录
2. 创建 `.py` 文件（例如 `my_mod.py`）
3. 编写节点代码
4. 重启编辑器，新节点会自动出现在左侧节点面板

### 2.2 编写第一个模组

在 `pyqt6_editor/mods/` 目录下创建 `my_mod.py`：

```python
"""
我的第一个 TACZ 模组
作者：玩家名
版本：1.0
功能：自定义节点
"""

# 导入必要的模块
from pyqt6_editor.node import Node, PortType
from pyqt6_editor.registry import register_node


# ==================== 自定义节点 ====================

@register_node("我的模组", "#FF5722", "回血", "恢复玩家生命值")
class HealNode(Node):
    """回血节点 - 恢复玩家生命值"""

    def _init_ports(self):
        # 添加输入端口（执行入口）
        self.add_input("入口", PortType.EXEC, "exec")
        # 添加输出端口（执行出口）
        self.add_output("出口", PortType.EXEC, "exec")

    def _get_config(self):
        # 可配置参数
        return {
            'heal_amount': 25,  # 回血量
        }

    def execute(self):
        # 执行逻辑（可选，根据需要实现）
        pass
```

## 3. 节点创建与注册

### 3.1 节点类结构

每个节点类必须包含以下部分：

```python
@register_node("分类名", "#颜色代码", "显示名称", "描述信息")
class MyNode(Node):
    """节点文档字符串"""

    def _init_ports(self):
        """初始化端口"""
        # 输入端口
        self.add_input("端口名", PortType.EXEC, "exec")
        # 输出端口
        self.add_output("端口名", PortType.EXEC, "exec")

    def _get_config(self):
        """返回配置字典"""
        return {'param_name': 'default_value'}

    def execute(self):
        """执行节点逻辑"""
        # 执行代码
        pass
```

### 3.2 注册装饰器

`@register_node` 装饰器参数说明：
- `tag` - 分类标签（中文），用于在左侧面板中对节点进行分组
- `color` - 节点背景颜色，使用十六进制颜色码（如 #FF69B4）
- `title` - 中文显示名称，节点左上角显示的文字
- `description` - 节点提示信息，鼠标悬停时显示的详细说明

### 3.3 端口类型

| 类型 | 说明 | 示例 |
|------|------|------|
| `PortType.EXEC` | 执行端口（流程控制） | 入口、出口 |
| `PortType.DATA` | 数据端口（传递数据） | 数值、字符串 |

### 3.4 数据类型

| 类型 | 说明 |
|------|------|
| `"exec"` | 执行流 |
| `"any"` | 任意类型 |
| `"number"` | 数字 |
| `"string"` | 字符串 |
| `"bool"` | 布尔值 |
| `"list"` | 列表 |

## 4. 高级节点开发

### 4.1 带数据处理的节点

```python
@register_node("数学运算", "#4CAF50", "加法", "两个数相加")
class AddNode(Node):
    """加法节点"""

    def _init_ports(self):
        self.add_input("输入1", PortType.DATA, "number")
        self.add_input("输入2", PortType.DATA, "number")
        self.add_output("结果", PortType.DATA, "number")

    def execute(self):
        # 获取输入数据
        input1 = self.get_input_by_name("输入1").get_data() or 0
        input2 = self.get_input_by_name("输入2").get_data() or 0
        # 计算结果
        result = input1 + input2
        # 设置输出数据
        self.get_output_by_name("结果").set_data(result)
```

### 4.2 带多个输出的节点

```python
@register_node("逻辑", "#2196F3", "条件分支", "根据条件选择执行路径")
class BranchNode(Node):
    """条件分支节点"""

    def _init_ports(self):
        self.add_input("入口", PortType.EXEC, "exec")
        self.add_input("条件", PortType.DATA, "bool")
        self.add_output("成立", PortType.EXEC, "exec")
        self.add_output("不成立", PortType.EXEC, "exec")
```

### 4.3 事件触发型节点

```python
@register_node("事件", "#9C27B0", "玩家受伤", "玩家受伤时触发")
class HurtNode(Node):
    """受伤事件节点"""

    def _init_ports(self):
        # 只有输出端口，没有输入端口（事件触发型）
        self.add_output("触发", PortType.EXEC, "exec")
        self.add_output("伤害值", PortType.DATA, "number")
```

## 5. 自定义连线类型开发

### 5.1 连线颜色配置

编辑器支持根据数据类型自动设置连线颜色。在节点类中，你可以通过以下方式指定端口的数据类型：

```python
def _init_ports(self):
    # 红色连线（例如错误类型）
    self.add_input("错误", PortType.DATA, "error")
    # 绿色连线（例如成功类型）
    self.add_output("成功", PortType.DATA, "success")
```

### 5.2 连线路由

编辑器内置了连线自动路由功能，可以自动绕开其他节点。你可以在编辑器设置中启用或禁用此功能。

## 6. 主题与样式定制

### 6.1 主题切换

编辑器支持暗色主题和亮色主题。你可以通过以下方式在模组中影响主题：

```python
# 在节点类中设置主题相关属性
class MyThemedNode(Node):
    # 暗色主题颜色
    dark_color = "#333333"
    # 亮色主题颜色
    light_color = "#FFFFFF"
```

### 6.2 自定义样式

你可以通过修改节点的绘制方法来自定义节点样式：

```python
def paint(self, painter, option, widget):
    # 自定义绘制逻辑
    super().paint(painter, option, widget)
    # 添加额外的绘制代码
```

## 7. 事件处理与交互实现

- `Ctrl+D` - 取消所有选中

## 8. 性能优化最佳实践

### 8.1 节点优化

1. **减少计算量**：在 `execute` 方法中避免复杂计算
2. **使用缓存**：对于重复计算的结果使用缓存
3. **批量处理**：对于多个类似操作，使用批量处理节点
4. **减少端口数量**：只添加必要的端口

### 8.2 渲染优化

1. **减少重绘**：避免频繁调用 `update()`
2. **使用合适的图形项**：根据需要选择合适的图形项类型
3. **优化绘制逻辑**：简化绘制代码，避免不必要的绘制

### 8.3 内存管理

1. **及时清理**：不再使用的资源及时清理
2. **避免内存泄漏**：注意循环引用问题
3. **合理使用数据结构**：选择合适的数据结构存储数据

## 10. 常见问题与解决方案

### 10.1 节点不显示在面板中

**原因**：
- 节点类没有使用 `@register_node` 装饰器
- 装饰器参数不正确
- 脚本没有被正确加载

**解决方案**：
- 确保使用 `@register_node` 装饰器
- 检查装饰器参数是否正确
- 检查脚本是否有语法错误

### 10.2 连接无法创建

**原因**：
- 端口类型不匹配（EXEC 端口只能连接到 EXEC 端口）
- 输入端口已经有连接（每个输入端口只能有一个连接）

**解决方案**：
- 确保端口类型匹配
- 先删除现有连接再创建新连接

### 10.3 模组加载失败

**原因**：
- 模组文件格式不正确
- 模组依赖缺失
- 模组代码有语法错误

**解决方案**：
- 检查模组文件格式
- 确保所有依赖都已安装
- 检查模组代码是否有语法错误

## 12. 示例模组

### 12.1 数学运算模组

```python
"""
数学运算模组
作者：TACZ 团队
版本：1.0
描述：提供常用数学运算节点
"""

from pyqt6_editor.node import Node, PortType
from pyqt6_editor.registry import register_node


@register_node("数学运算", "#4CAF50", "加法", "两个数相加")
class AddNode(Node):
    def _init_ports(self):
        self.add_input("输入1", PortType.DATA, "number")
        self.add_input("输入2", PortType.DATA, "number")
        self.add_output("结果", PortType.DATA, "number")

    def execute(self):
        a = self.get_input_by_name("输入1").get_data() or 0
        b = self.get_input_by_name("输入2").get_data() or 0
        self.get_output_by_name("结果").set_data(a + b)


@register_node("数学运算", "#2196F3", "减法", "两个数相减")

        self.add_input("被减数", PortType.DATA, "number")
        self.add_input("减数", PortType.DATA, "number")
        self.add_output("结果", PortType.DATA, "number")

    def execute(self):
        a = self.get_input_by_name("被减数").get_data() or 0
        b = self.get_input_by_name("减数").get_data() or 0
```

### 12.2 字符串处理模组

```python
"""
字符串处理模组
作者：TACZ 团队
版本：1.0
描述：提供字符串处理节点
"""

from pyqt6_editor.node import Node, PortType
from pyqt6_editor.registry import register_node


@register_node("字符串", "#FF9800", "连接", "连接两个字符串")
class ConcatNode(Node):
    def _init_ports(self):
        self.add_input("字符串1", PortType.DATA, "string")
        self.add_input("字符串2", PortType.DATA, "string")

    def execute(self):
        s1 = self.get_input_by_name("字符串1").get_data() or ""
        s2 = self.get_input_by_name("字符串2").get_data() or ""
        self.get_output_by_name("结果").set_data(s1 + s2)


@register_node("字符串", "#9C27B0", "长度", "获取字符串长度")
class LengthNode(Node):
    def _init_ports(self):
        self.add_input("字符串", PortType.DATA, "string")
        self.add_output("长度", PortType.DATA, "number")

    def execute(self):
        s = self.get_input_by_name("字符串").get_data() or ""
        self.get_output_by_name("长度").set_data(len(s))
```
